> **A general-purpose knowledge distillation framework for stereo 3D reconstruction foundation models.**
> Compress any MASt3R-style teacher into a lightweight student — applicable to any domain, any dataset.
> We demonstrate it on two settings: specialized distillation for lunar imagery (MOONSt3R teacher)
> and generalist distillation on BlendedMVS (standard MASt3R teacher).
> A broader extension to additional domains and architectures is currently in preparation.
>
> The **teacher** is [MASt3R](https://github.com/naver/mast3r), a 688M-parameter geometric
> foundation model. **Students** are compact networks (CNN or ViT encoder + lightweight
> cross-view decoder) trained entirely from teacher pseudo-GT — no ground-truth annotations required.
> The framework provides: SVD-based decoder initialization, cosine-margin feature alignment loss,
> and a multi-student training loop where all students share one teacher forward pass per iteration.

## Framework Overview

```
Teacher (MASt3R, frozen)
        |
        |  one forward pass per batch
        v
   Pseudo-GT (pts3d, descriptors, confidence)
        |
        +---> Student 1 (MobileNetV3 encoder + lightweight decoder)
        |          geometry loss + [feature alignment] + [gradient loss]
        |
        +---> Student 2 (DINOv2 ViT-S encoder + lightweight decoder)
        |          geometry loss + [feature alignment] + [gradient loss]
        |
        +---> Student N  ...  (arbitrary architecture)
```

All students share a single teacher forward pass per iteration.  Each student
is trained independently (separate optimizer, separate checkpoint) with two
optional loss components:

1. **Geometry loss** (output-level): `ConfLoss(Regr3D)` + `ConfMatchingLoss` applied
   to the pseudo-GT generated by the teacher.  This is the standard MASt3R loss
   and requires no architectural changes.

2. **Feature alignment loss** (optional, `--kd_feat_weight`): aligns encoder
   features between student and teacher using a cosine-margin loss (Depth
   Anything V2, Eq. 9).  A learnable per-student projector handles dimension
   mismatches and is trained jointly with the encoder.

**SVD initialization** (`--svd_init`): warm-starts the student decoder and
heads from the teacher weights by computing a truncated SVD projection.  This
gives an optimal low-rank approximation in the Frobenius-norm sense and
significantly reduces the number of epochs needed to reach good performance.

## Reproducing Paper Students

The four students evaluated in the paper can be reproduced with the commands below.
All share the same teacher (`MOONSt3R.pth`), dataset, and training recipe.

| ID | Encoder | Decoder (dim/depth/heads) | Params | Role |
|----|---------|--------------------------|--------|------|
| S1 | MobileNetV3-Large | 512 / 6 / 4 | 135.7 M | CNN baseline |
| S2 | DINOv2 ViT-S/14 | 512 / 6 / 4 | 154.9 M | Best student |
| S3 | DINOv2 ViT-S/14 | 256 / 4 / 4 | 94.9 M | Reduced decoder (7.3× smaller than teacher) |
| S4 | ViT-Tiny/16 | 512 / 6 / 4 | 138.5 M | Smaller ViT encoder |
| Teacher | ViT-Large | 768 / 12 / 12 | 688.6 M | MASt3R (frozen) |

```bash
export TEACHER_CKPT=/path/to/MOONSt3R.pth
export DATA_ROOT=/path/to/StereoLunar
export OUTPUT_DIR=/path/to/output

# S2 — best student (DINOv2 ViT-S, full decoder, feature alignment + SVD init)
torchrun --standalone --nproc-per-node=7 distillation_dual.py \
  --teacher_ckpt $TEACHER_CKPT \
  --train_dataset "LunarDataset(split='train', ROOT='${DATA_ROOT}/nadir_clean_0_90', ...)" \
  --s2_output_dir ${OUTPUT_DIR}/s2_vit_small \
  --s2_dec_embed_dim 512 --s2_dec_depth 6 --s2_dec_heads 4 --s2_prefer_dinov2 \
  --disable_s1 --disable_s3 --disable_s4 --disable_s5 \
  --kd_feat_weight 0.1 --kd_feat_mode cosine_margin --kd_feat_margin 0.9 \
  --svd_init --teacher_dec_depth 12 \
  --lr 3e-5 --epochs 50 --batch_size 1

# S3 — reduced decoder (same encoder as S2, decoder 256/4/4)
torchrun --standalone --nproc-per-node=7 distillation_dual.py \
  --teacher_ckpt $TEACHER_CKPT \
  --s4_output_dir ${OUTPUT_DIR}/s3_reduced \
  --s4_dec_embed_dim 256 --s4_dec_depth 4 --s4_dec_heads 4 --s4_prefer_dinov2 \
  --enable_s4 --disable_s1 --disable_s2 --disable_s3 --disable_s5 \
  --kd_feat_weight 0.1 --kd_feat_mode cosine_margin --kd_feat_margin 0.9 \
  --svd_init --teacher_dec_depth 12 \
  --lr 3e-5 --epochs 50 --batch_size 1

# S1 — MobileNet CNN baseline
torchrun --standalone --nproc-per-node=7 distillation_dual.py \
  --teacher_ckpt $TEACHER_CKPT \
  --output_dir ${OUTPUT_DIR}/s1_mobilenet \
  --s1_dec_embed_dim 512 --s1_dec_depth 6 --s1_dec_heads 4 \
  --s1_backbone_name mobilenetv3_large_100 \
  --disable_s2 --disable_s3 --disable_s4 --disable_s5 \
  --kd_feat_weight 0.1 --kd_feat_mode cosine_margin --kd_feat_margin 0.9 \
  --svd_init --teacher_dec_depth 12 \
  --lr 3e-5 --epochs 50 --batch_size 1

# S4 — ViT-Tiny encoder
torchrun --standalone --nproc-per-node=7 distillation_dual.py \
  --teacher_ckpt $TEACHER_CKPT \
  --s5_output_dir ${OUTPUT_DIR}/s4_vit_tiny \
  --s5_dec_embed_dim 512 --s5_dec_depth 6 --s5_dec_heads 4 \
  --s5_model_name vit_tiny_patch16_224 \
  --enable_s5 --disable_s1 --disable_s2 --disable_s3 --disable_s4 \
  --kd_feat_weight 0.1 --kd_feat_mode cosine_margin --kd_feat_margin 0.9 \
  --svd_init --teacher_dec_depth 12 \
  --lr 3e-5 --epochs 50 --batch_size 1
```

> All students can also be co-trained in a single run (one teacher forward pass serves all).

## Defining a Custom Student

A student is an `AsymmetricMASt3R` instance whose built-in ViT encoder has been
replaced by a lightweight backbone.  The minimal interface is:

```python
# distillation/students.py — minimal pattern for a new student

import types
import torch.nn as nn
from mast3r.model import AsymmetricMASt3R
from distillation.students import _build_mast3r_shell, _make_encode_image

class MyBackbone(nn.Module):
    """Your custom backbone — outputs [B, dec_embed_dim, H/patch, W/patch]."""
    def __init__(self, out_dim: int):
        super().__init__()
        # ... define layers ...
        self.patch = 16  # spatial stride

    def forward(self, x):
        # ... compute features ...
        return feat  # [B, out_dim, H/16, W/16]


def build_my_student(device='cuda', dec_embed_dim=512, dec_depth=6,
                     dec_num_heads=4, mlp_ratio=1.0):
    # 1. Build the decoder + head scaffold (enc_depth=0, uses your backbone instead)
    student = _build_mast3r_shell(device, dec_embed_dim, dec_depth, dec_num_heads, mlp_ratio)

    # 2. Attach your backbone
    student.tiny_enc = MyBackbone(out_dim=dec_embed_dim).to(device)

    # 3. Bind the encoder method (converts [B,D,H,W] feature map -> tokens + positions)
    student._encode_image = types.MethodType(
        _make_encode_image(patch_size=16, feat_attr='tiny_enc'), student)

    return student
```

Then register the student in `distillation_dual.py`:

```python
# In main(), after the existing student blocks:
if getattr(args, 'enable_my_student', False):
    my_student = build_my_student(device=device, dec_embed_dim=args.my_dec_embed_dim, ...)
    students_info.append(('S_My', my_student, args.my_dec_depth,
                          args.my_output_dir, args.my_weight))
```

The student will automatically participate in the shared teacher forward pass,
receive pseudo-GT, and be optimized with the full loss suite.

## SVD Initialization

SVD init warm-starts the student decoder from the teacher's decoder weights by
computing a truncated SVD projection into the student's smaller dimensional space.
For a teacher weight matrix W_T in R^(d_T x d_T):

```
W_T = U diag(sigma) V^T
W_S = U[:d_S, :r] diag(sigma[:r]) V^T[:r, :d_S]    where r = min(d_S, rank(W_T))
```

This is the optimal rank-r approximation (Eckart-Young theorem).

```python
from distillation.svd_init import svd_init_student_from_teacher

# Load teacher state dict
teacher_sd = {k: v.cpu() for k, v in teacher.state_dict().items()}

# Initialize student decoder from teacher
svd_init_student_from_teacher(
    student=my_student,
    teacher_state=teacher_sd,
    student_dec_depth=6,     # decoder blocks in student
    teacher_dec_depth=12,    # decoder blocks in teacher
    layer_mapping_strategy='uniform',  # 'uniform' | 'last_k' | 'first_k'
    compression_mode='svd',            # 'svd' | 'direct_truncation'
    verbose=True,
)
```

Enable from the command line: `--svd_init --teacher_dec_depth 12`

## Feature Alignment Loss

The cosine-margin loss (Depth Anything V2, Eq. 9) aligns encoder features
between student and teacher.  Pixels that are already well-aligned
(cosine similarity >= alpha) are excluded, focusing capacity on hard regions.

```python
from distillation.feature_loss import DepthAnythingFeatureAlignLoss

feat_loss = DepthAnythingFeatureAlignLoss(alpha=0.9)

# During training (inside distillation_dual.py):
# feat_s: [B, C_s, H, W]  student encoder features (requires_grad=True)
# feat_t: [B, C_t, H, W]  teacher encoder features (detached)
loss_feat = feat_loss(feat_s, feat_t, student_name='S2_ViT-Small')
```

When C_s != C_t, a per-student learnable projector (C_s -> C_t) is created
automatically and added to the student's optimizer parameter group.

Enable from the command line:
```
--kd_feat_weight 0.1 --kd_feat_mode cosine_margin --kd_feat_margin 0.9
```

## Usage Example 1 — StereoLunar (Specialized Distillation)

**Teacher:** MOONSt3R — MASt3R fine-tuned on the StereoLunar dataset, adapted to
lunar orbital imagery (low texture, extreme illumination, limited parallax).
Students are trained to specialize in this domain from the adapted teacher's
dense pseudo-GT, without requiring ground-truth depth or pose annotations.

### Dataset structure

The StereoLunar dataset uses the MASt3R dataset format.  Each subdirectory
contains one acquisition scenario (nadir/pitch/landing) and must be
pre-processed into `(image1, image2, pts3d_gt, intrinsics)` pairs stored in
a `pairs.npz` file with train/test split files.

```
$DATA_ROOT/
  nadir_clean_0_90/
    train.npz        # pair index (train split)
    test.npz         # pair index (test split)
    images/
      im_00000.jpg
      im_00001.jpg
      ...
    depth/
      im_00000.exr
      ...
    cameras/
      im_00000.npz   # intrinsics + cam2world
      ...
  nadir_clean_90_180/
  nadir_clean_180_270/
  pitch_clean_0_90/
  pitch_clean_90_180/
  pitch_clean_180_270/
  pitch_clean_ajout/
  landingimages/
```

### Dataloader

```python
from mast3r.datasets import LunarDataset

dataset = LunarDataset(
    split='train',
    ROOT='$DATA_ROOT/nadir_clean_0_90',
    split_file='train.npz',
    resolution=512,
    n_corres=8192,
    nneg=0.5,
    transform='ColorJitter',
    aug_crop='auto',
)
```

### Training

```bash
export TEACHER_CKPT=/path/to/MOONSt3R.pth
export DATA_ROOT=/path/to/StereoLunar
export OUTPUT_DIR=/path/to/output/moon_experiment

bash scripts/train/train_moon_example.sh
```

Key flags used in the Moon experiment:
```
--svd_init --teacher_dec_depth 12
--kd_feat_weight 0.1 --kd_feat_mode cosine_margin --kd_feat_margin 0.9
--teacher_conf_thresh 0.2
--lr 3e-5 --epochs 60
```

### Evaluation

```bash
# Evaluate a trained student on the Moon test set
python evaluation/moon/evalAllPairs.py \
    --model_path "${OUTPUT_DIR}/s2_vit_small/checkpoint-50.pth" \
    --test_root "${DATA_ROOT}/nadir_clean_0_90" \
    --output_dir results/moon/

# Aggregate metrics across all pairs and scenarios
python compare_pair_reports.py --root results/moon/
```

## Usage Example 2 — BlendedMVS (Generalist Distillation)

**Teacher:** standard MASt3R (`MASt3R_ViTLarge_BaseDecoder_512_catmlpdpt_metric.pth`) —
a general-purpose 3D foundation model, not domain-adapted.  This experiment shows that
the same distillation pipeline transfers to a generalist setting.

**Key findings on this benchmark:**
- **SVD initialization** consistently improves convergence and final performance across all students.
- **Feature alignment loss** helps when the encoder is fine-tuned; it can slightly hurt when the encoder is frozen (adapter-only training).
- **Encoder adaptation is critical**: freezing the DINOv2 backbone leads to a significant performance drop, confirming that the pretrained features must be adapted even when starting from a strong initialization.
- **Decoder compression is safe**: reducing the decoder from 512/6 to 256/4 incurs only marginal degradation, while halving the decoder parameter count.

> 📄 A follow-up paper extending these findings to additional datasets and architectures is currently in preparation.

### Dataset structure

BlendedMVS should be pre-processed into the MASt3R dataset format.

```
$DATA_ROOT/              # blendedmvs_processed/
  scene_0000/
    pairs.npz            # pair index for the scene
    images/
      000000.jpg
      000001.jpg
      ...
    depth/
      000000.exr
      ...
    cameras/
      000000.npz
      ...
  scene_0001/
  ...
```

### Dataloader

```python
from dust3r.datasets import BlendedMVS

dataset = BlendedMVS(
```

## Key Training Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--teacher_ckpt` | required | Path to teacher checkpoint (.pth) |
| `--svd_init` | off | Warm-start student decoders from teacher via truncated SVD |
| `--teacher_dec_depth` | 12 | Number of decoder blocks in the teacher (for layer mapping) |
| `--svd_layer_mapping` | uniform | Layer mapping strategy: `uniform` / `last_k` / `first_k` |
| `--kd_feat_weight` | 0.0 | Weight for feature alignment loss (0 = disabled) |
| `--kd_feat_mode` | mixed | Feature loss mode: `cosine_margin` (recommended) or `mixed` |
| `--kd_feat_margin` | 0.9 | Tolerance margin alpha for cosine_margin mode |
| `--teacher_conf_thresh` | 0.0 | Confidence threshold for pseudo-GT filtering |
| `--s{N}_freeze_backbone` | off | Freeze student N's encoder (only adapter + decoder train) |
| `--disable_s{N}` | off | Skip student N entirely |
| `--enable_s{N}` | off | Enable optional student N (S3-S11) |
| `--no_feat_students` | "" | Comma-separated student names to exclude from feature loss |
| `--no_svd_students` | "" | Comma-separated student names to skip SVD init |
| `--resume` | off | Resume from checkpoint-last.pth in each output_dir |
